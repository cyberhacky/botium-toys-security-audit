# Botium Toys Security Audit & Compliance Assessment

## Overview

This project demonstrates a simulated internal cybersecurity audit conducted for Botium Toys, a fictional retail company experiencing rapid online business growth.

The audit was performed to evaluate the organization’s current security posture, identify risks and vulnerabilities, assess existing security controls, and review compliance with industry standards and regulations.

This assessment aligns with the following frameworks and standards:

- NIST Cybersecurity Framework (NIST CSF)
- PCI DSS
- GDPR
- SOC 1 / SOC 2

---

# Audit Scope

The scope of this audit included the review of:

- Employee devices and workstations
- Internal network infrastructure
- Ecommerce systems
- Data storage and retention systems
- Legacy systems
- Physical security controls
- Access management practices
- Security monitoring procedures

The objective was to identify security gaps that could impact confidentiality, integrity, availability, compliance, and business continuity.

---

# Risk Assessment Summary

Botium Toys currently faces several security risks due to incomplete controls and weak compliance practices.

### Key Risks Identified

- Excessive employee access to sensitive data
- Lack of encryption for customer payment information
- No intrusion detection system (IDS)
- Missing disaster recovery and backup procedures
- Weak password management practices
- Undefined legacy system maintenance procedures
- Inadequate access control implementation

### Overall Risk Rating

**Risk Score: 8/10 (High Risk)**

The organization is exposed to potential:

- Data breaches
- Compliance violations
- Financial penalties
- Operational disruption
- Unauthorized access to customer data

---

# Recommendations

To improve the organization’s security posture, the following actions are recommended:

- Implement least privilege access controls
- Deploy an intrusion detection system (IDS)
- Encrypt sensitive customer and payment data
- Establish disaster recovery and backup procedures
- Implement centralized password management
- Strengthen password complexity requirements
- Improve asset inventory and classification
- Create formal legacy system maintenance procedures
- Conduct regular security audits and risk assessments
